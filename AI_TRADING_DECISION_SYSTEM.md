# AI売買判断システム設計

前提日：2026-06-17

## 目的

AIを「当たり銘柄を予想する道具」ではなく、売買で勝ちを得るための方法論、日々の外的要因、銘柄ごとの値動き、見送り理由を蓄積し、最終判断の質を上げるための外部記憶システムとして使う。

本番売買に備え、1か月の模擬運用で以下を蓄積する。

- 勝ちやすい買いタイミング
- 利確・損切りの実効性
- 見送りが正解だった条件
- 政治・政策・金利・為替・業界材料と値動きの関係
- AIごとの判断観点と弱点

## 基本思想

1. AIに売買を断定させない。最終判断はユーザーが行う。
2. AIには、候補、根拠、反対理由、損益比、外的要因を整理させる。
3. 勝率だけでなく、平均利益、平均損失、見送りによる損失回避を評価する。
4. 価格・ニュース・政治・政策・業界材料は、取得日と出典を残す。
5. 事実、推測、仮説、判断を分けて記録する。

## AIに収集させる方法論

| 領域 | 集めるノウハウ |
|---|---|
| ろうそく線 | 下ヒゲ陽線、上ヒゲ、包み足、連騰後の陰線、窓開け後の失速 |
| トレンド | 高値更新、押し目反発、移動平均との位置、出来高増減 |
| 資金管理 | 初回投入額、分散、現金余力、損切り幅、1回あたり許容損失 |
| 利確 | 第1利確、第2利確、半分利確、上ヒゲ後の利確 |
| 損切り | 終値割れ、監視ライン割れ、材料否定、想定外の大陰線 |
| 見送り | 高値追い、損益比の悪化、材料不明、値動きが荒すぎる銘柄 |
| 検証 | 翌日検証、5営業日検証、10回単位の勝率・損益比 |

## 外的要因の記録軸

| 外的要因 | 影響しやすい銘柄・業界 | 見るポイント |
|---|---|---|
| 金利 | 銀行、保険、不動産、成長株 | 利ざや改善、金利上昇メリット、割高株の調整 |
| 為替 | 輸出、半導体、機械 | 円安メリット、円高による利益圧迫 |
| 政府政策 | 防衛、通信、半導体、エネルギー | 予算、規制、補助金、国策テーマ |
| 選挙・政局 | 全体相場、政策関連株 | 政策期待、リスク回避、地合い変化 |
| 米国市場 | 半導体、AI、ソフトバンクG | Nasdaq、SOX指数、米金利、AI関連ニュース |
| 業界ニュース | 各セクター | 決算、受注、設備投資、競合ニュース |
| 地合い | 全銘柄 | 日経平均、TOPIX、売買代金、全面高/全面安 |

## CoworkとCodexの役割分担

| 役割 | Cowork | Codex |
|---|---|---|
| 日々の対話 | ユーザーと相場観、候補、仮説を整理 | 必要に応じてローカルファイルへ反映 |
| 外部情報収集 | 価格、ニュース、政治・政策・業界材料を確認 | 取得データを構造化・ログ化 |
| 判断補助 | 売買フラグ、見送り理由、反対意見を出す | ルール化、テンプレート化、検証軸の整備 |
| 外部記憶 | 新規メモ作成、会話の流れを残す | 既存mdの更新、Git反映、管理ファイル更新 |
| 1か月後の統合 | 学びと相場背景をまとめる | 勝率・損益比・ルール案として整理 |

## 判断時の標準プロンプト

日次チェックでは、AIに以下の順で考えさせる。

1. 今日の地合いは追い風か逆風か。
2. 銘柄ごとのろうそく線は、買い、売り、待ち、見送りのどれか。
3. その判断を支える外的要因はあるか。
4. 逆に、判断を否定する材料はあるか。
5. 損切りまでの距離と利確までの距離は釣り合うか。
6. 今日入る理由と、今日入らない理由のどちらが強いか。
7. 明日検証すべき条件は何か。
