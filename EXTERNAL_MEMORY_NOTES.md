# 外部記憶化ノウハウ（AI-Hubの手法を抽出してトレーディングに適用）

最終更新：2026-06-16　前提日：2026-06-16

## このファイルの位置づけ

AI-Hubフォルダで確立されている「記憶の外部化（コンテキストをDriveに残し、複数AIで引き継ぐ）」の**手法・構造だけ**を抽出し、トレーディングフォルダの外部記憶化に活かすためのノウハウ集。

> 独立運用の維持：AI-Hubの各プロジェクトの**中身（内容）は持ち込まない**。借りるのは構造・命名・更新プロトコルのみ。トレーディングは引き続きAI-Hubと独立。出典＝AI-Hub `_MASTER/` `_TEMPLATES/` 等を2026-06-16に手法目的で参照。

---

## A. AI-Hubから抽出した外部記憶化の要点

1. **入口ファイルと読む順序の固定**：`ENTRY.md` が「全AIはまずこれを読む→次は○○」と読む順番だけ規定。ルール本体は各ファイルに置き、入口には転記しない。
2. **「現在」と「履歴」の二層分離**：`PROGRESS.md`＝現在状態の唯一の正（毎回上書き）／`_PROGRESS_LOG/`＝日付ファイルで履歴を蓄積。
3. **記憶を3層に分ける**：不変の前提（CONTEXT）／背景の設計・判断（knowledge）／現状と数値（進捗・数値）。性質の違うものを混ぜない＝ハルシネーション低減。
4. **セッション終了プロトコルの自動化**：「今日はここまで」等のトリガー語で、サマリー→現状更新→日付ログ作成→マップ確認→他AI引き継ぎプロンプト作成、を定型実行。
5. **命名規則と上書き制約への対応**：`YYYY-MM-DD_AI名_session.md`、UTF-8/LF、ASCII+アンダースコア。Drive MCPは上書き不可なので新規作成で代替、古い版はユーザーが手動削除、ファイルIDを記録。
6. **正本の唯一性**：「一覧の唯一の正」を一箇所に決め、他ファイルに表を重複させない。
7. **引き継ぎの現実認識**：引き継げる＝ルール・成果物・進捗／引き継げない＝会話のトーン・判断の経緯。だから「決まったことを都度書き足す」。

## B. トレーディングへの適用

### すでに合致している点

- 二層分離：`TRADING_MAP.md`＝現在の正／`LEARNING_LOG.md`＝履歴の蓄積。
- 命名規則：`candlestick_charts_20260614/` `trading_sim_20260614/` など前提日を名前に付与済み。
- 正本の唯一性：状態・構成の表はTRADING_MAPのみ（他に重複させない）。

### 追加で取り入れる4点

| 借りる手法 | トレーディングでの適用 |
|---|---|
| 入口＋読む順序 | TRADING_MAP冒頭に「AIはまず TRADING_MAP → 次に LEARNING_LOG を読む」と明記 |
| 3層分離 | 不変の前提（資金・ルール・禁止事項）／型と背景（学び・上昇背景）／現状と数値（価格・監視ライン）を見出しで分ける |
| 終了プロトコル | 「今日はここまで」等のトリガーで LEARNING_LOG に追記する手順を運用化 |
| 上書き制約の明文化 | 「新規作成はDriveコネクタ、既存.mdの編集・Git操作はMac側」をルールに記載（恒久化） |

### トレーディングの推奨ファイル構成（外部記憶化版）

```
トレーディング/
├── TRADING_MAP.md       … 入口＋現在状態の唯一の正（まず読む）
├── LEARNING_LOG.md      … セッション履歴の蓄積（次に読む）
├── trading_policy_*.md  … 不変寄りの前提（資金管理・損切利確の方針）
├── EXTERNAL_MEMORY_NOTES.md … 本ファイル（外部記憶化の運用ノウハウ）
├── candlestick_charts.html  … ライブチャート
└── outputs/
    ├── candlestick_charts_YYYYMMDD/ … チャート画像（前提日つき）
    └── trading_sim_YYYYMMDD/        … シミュレーション
```

## C. セッション終了プロトコル（トレーディング版・運用案）

トリガー語：「今日はここまで」「終了」「おわり」など。

1. 当日サマリー生成（実施したこと／判明したこと／持ち越し）。
2. `LEARNING_LOG.md` に「## セッション YYYY-MM-DD（前提日 YYYY-MM-DD）」を追記（価格は出典・概算を明記）。
3. `TRADING_MAP.md` の現在状態・更新履歴を確認（変更があればMac側で反映）。
4. 必要なら他AI（Codex等）への引き継ぎプロンプトを別ファイルで作成。

> 制約メモ：このCoworkセッションでは既存.mdの上書きがブロックされる場合がある。その際は新規ファイル作成（コネクタ）で代替し、既存ファイルへの反映はMac側で行う。古い版が溜まったらユーザーが手動削除。
